API structure

Edit API

GET /

Health check. Returns service status and version.

Response: status, version.

POST /videos/upload

Uploads a video file and stores probed metadata.

Request: multipart file.

Response: id, original filename, stored path, size, duration, mime type, upload time.

GET /videos

Lists uploaded videos with pagination.

Query: page, page_size.

Response: items, total, page, page_size, has_next, has_prev.

POST /trim

Creates a trim job from an uploaded video or source variant.

Request: video_id, start, end, optional source_variant_id.

Response: job_id.

POST /overlays

Creates an overlay job with text, image, video and optional watermark layers.

Request: video_id, optional source_variant_id, overlays, optional watermark.

Response: job_id.

POST /overlays/watermark

Creates a full-duration watermark job.

Request: video_id, optional source_variant_id, watermark.

Response: job_id.

POST /variants/transcode/{video_id}

Creates a multi-quality transcode job.

Request: qualities such as 1080p, 720p, 480p.

Response: job_id.

GET /variants/videos/{video_id}/variants

Lists all generated variants for a video with pagination.

Query: page, page_size.

Response: items, total, page, page_size, has_next, has_prev.

GET /variants/{variant_id}/download

Downloads a generated variant file.

Response: file stream.

GET /jobs/status/{job_id}

Returns the current status and progress for a background job.

Response: job_id, status, progress, error_message.

GET /jobs/result/{job_id}

Streams the completed job output when the job succeeded.

Response: file stream or job state error.
