Edit API

Minimal video processing backend for uploads, trims, overlays, watermarks and transcodes.
Built as a compact API layer for editor tools and media workflows.

Tech API structure

Tech used

FastAPI for the HTTP API and OpenAPI docs.

FFmpeg for trim, overlay, watermark and transcode work.

Postgres for videos, variants and job state.

SQLAlchemy for database access.

Alembic for schema migrations.

Docker for the API, database and local services.

API structure

/videos uploads files and stores video metadata.

/trim creates a clipped variant from a time range.

/overlays applies text, image, video and watermark overlays.

/variants lists processed outputs for a source video.

/jobs exposes job status and processing results.

How it works

A client uploads a video, requests an operation, then polls a job until the generated variant is ready.

Stored job states include pending, started, success and failure. Output files are grouped as uploads, variants and processed transcodes.
